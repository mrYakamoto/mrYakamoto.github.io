Enumerable's Enumerator, group_by

What are enumerator and enumerable? Enumerable is a module that provides many methods of searching, traversing, and altering collections objects, one element at a time. It's already included in some classes like Array and Hash, and it can be added as a "mixin" to any class with a defined .each method. These methods included in Enumerable for going through elements one at a time are enumerators, which basically means just that, going through things one at a time.

One such enumerator in the Enumerable module is the method group_by. group_by passes each element in a collection through a code block and sorts the elements by the result. A Hash is created, and each unique result that's generated by the code block becomes a key in that Hash. The original element that groupby sent to the code block becomes a value in the key that it generates. This can be expressed much more clearly in an example. 

array = [3,4,5,6,7,8,9,10]
array.group_by {|i| i % 3}

Results in: 
{ 0=>[3, 6, 9], 1=>[4, 7, 10], 2=>[5, 8] }

Running group_by on a range would have the exact same result as the array. 
p (3..10).group_by {|i| i % 3}

also becomes... 
{ 0=>[3, 6, 9], 1=>[4, 7, 10], 2=>[5, 8] }

group_by can be applied in a ton of useful ways to easily sort collection elements into a hash of tight groups. Some other simple examples could be sorting an array of strings by the letter they start with or by their string.length.
