Ruby Classes

Classes are used extensively in Ruby language, and creating your own classes can save a lot of time and headache. Say you need to create several objects that are all going to share the same properties and methods. By creating a class, you only have to type all that code once. Every time you initialize a new class object, it'll already have all that code built in.

Here's an example of a Class called 'Kid' 

class Kid

  def initialize(name, favorite_color, height_inches)
    @name = name
    @favorite_color = favorite_color
    @height = height_inches
  end

  attr_reader :name
  attr_writer :favorite_color
  attr_accessor :height

  def hello
    puts "Hello #{@name}!"
  end

  def color
    puts "#{@name}'s favorite color is #{@favorite_color}"
  end

  def grow(added_height)
    @height += added_height
    puts "#{@name} grew #{added_height} inches. #{@name}'s now #{@height} inches tall!"
  end

end

So, what's going on here? First, if you'd like to be able to create object instances of this class, you need to write a method called initialize. This method is always called initialize inside of a class, and it's used to create a new instance of the class. Looks like there're three things that we want to keep track of inside the class Kid and that's name, favorite color, and height. If we were to create a new instance of Kid, it would look like this. 

boy = Kid.new("Charles", "green", 50)
p boy
# => <Kid:0x000000017c3f70 @name="Charles", @favorite_color="green", @height=50>

And there you have it. We've created a new object assigned to the variable 'boy.' 'boy' has a unique object number and the three variables that we used as arguments for the initialize method have been stored in variables inside the object. Not just any variables though, variables with an @ at the beginning are called instance variables. They're variables which are created in the class Kid code block. But, by making them instance variables, they're assigned to and accessible from a place that would normally be outside of their scope, in the newly created class object.

There's also some methods included. The methods #hello, #color, and #grow are written inside the class Kid. Thus, they are accessible in instances of that class. One of which we just so happen to have already created, stored in the variable 'boy.' Let's call those methods and see what they do. 

puts boy.name
# => Charles

'attr_reader' is a little shortcut in the class which allows the included variable to be accessed inside 'boy' using dot notation. We see that used here, and it returns simply the value of '@name', "Charles." 

boy.hello
# => Hello Charles!

Here we can see that the method #hello is included inside the object boy and can be called from there. 

boy.grow(5)
# => Charles grew 5 inches. Charles's now 55 inches tall!

The method grow is similarly included inside boy. This method requires one argument and uses that argument to permanently alter the instance variable height, stored in boy. In this case, it changes it from 50 to 55. 

boy.height = 30
puts boy.height
# => 30

'attr_writer' is another shortcut that allows for the user to alter the included variable's value through dot notation. In this class, there's an 'attr_writer' for height. So, by using 'boy.height = 30', the value of 'boy.height' has been permanently changed to 30. 

boy.favorite_color = "red"
boy.color
# => Charles's favorite color is red

'attr_accessor' includes both abilities of 'attr_reader' and 'attr_writer', so in this example, the variable height can be returned or altered in the same way as the previous examples. 

boy.name = "Gary"
# => @ line 39 :  
# => 39: boy.name = "Gary" 
# => NoMethodError: undefined method `name=' for #<Kid:0x000000023fd7a0> 
# => from solution.rb:39:in `<main>'

The variable name only has an attr_reader included for it. So, in the final line, when we try to alter the value of 'boy.name' like we would with an 'attr_writer' or 'attr_accessor', we get a NoMethodError. 'attr_reader' only enables the user to return that variable's value.

We could now go on to create as many instances of the class 'Kid' as we would like. We could quickly create objects for every kid in a classroom, and every 'Kid' object would have all the class methods instantly built into them. If we decided that we'd like to change one of those methods, or add some, we could do so in the class definition and it those changes would automatically be accessible through all class objects.

I almost forgot to talk about instance methods. I'll quickly explain them here. All the methods included in the previous example are instance methods, they exist in the instance of the object and have acess to all instance variables. Class methods however, exist only in the class and don't have acess to instance variables or methods. A class method is defined with self before the method name like such. 

def self.print_line
    puts "This class method prints this line"
end

A call to this method from the object 'boy' will yield a NoMethodError. As you can see, the method can be called from the class Kid though. 

boy.print_line
# => NoMethodError: undefined method `print_line' for#<Kid:0x00000001f036a0> 
Kid.print_line
# => This class method prints this line
