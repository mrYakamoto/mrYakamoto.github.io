Looping in Ruby vs.
Looping in JavaScript

While switching back and forth between Ruby and JavaScript, it becomes clear that there are some major distinctions between the two languages. How each language treats looping is a quality that can be easily used to demonstrate their differences. Ruby seems to be much more of a colloquially written language, more easily understood by even non-programmers. JavaScript has much stricter "grammar" and is written more with the computer in mind than the coder. Ruby has many built-in methods for the purpose of looping/iterating through collections. The core of these methods is #each.

#each is a built-in method in Ruby which is used to move through a collection and apply each element, one by one, to a given code block. The enumerable mix-in in Ruby provides a large selection of methods used to iterate through collections with all sorts of purposes in mind.

However, JavaScript provides no such built-in method for iterating over a collection. The foundation of iteration in JavaScript is the for/while loop. A for loop is often used for the identical purpose that #each would be used in Ruby, but in JavaScript each step of it must be written out for the program. 

for ( var i = 0; i < array.length; i++ ) {
code block here;
};

Here, the additional logic of initiating a counter and telling it to increment itself by a given value as long as the given boolean remains true, must be all written out. If we were using this for loop to iterate an array, the counter would have to be used as the index point for the desired elements.

While the syntax is dramatically different between Ruby and JavaScript, the core concepts are identical. JavaScript merely requires the process to be spelled out step by step while Ruby offers some built in shortcuts.
