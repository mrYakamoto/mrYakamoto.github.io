Arrays and Hashes

Arrays and hashes are very similar, but also have very distinct uses. In Ruby, everything is an object, and arrays and hashes are collections of objects.

In an array, each item contained can be almost any kind of object, like strings, numbers, or even other arrays.

Here's some examples of arrays...

array1 = [1,5,4,2,80]
array2 = ['clown', 'box','tar','cat']
array3 = [56, 'pencil', ['work', 41, 'gauntlet'], '99']

Each item in an array's position is marked in order by an index number (starting with zero), like array_name[indexNumber].

For example, 

array3[0] = 56
array3[2] = ['work', 41, 'gauntlet']
array3[2][0] = 'work'

Hashes are very much like arrays, except that each entry contains a pair of items, a "key" and a "value". The key points to the value and each key within the same hash must be unique.

A hash looks like this

hash_name = {
  "age" => 58,
  "address" => "451 Yorkshire Ln.",
  "occupation" => "Bible salesman"
}

The values in the hash can be accessed by using the keys. 

hash["age"] = 58
hash["address"] = "451 Yorkshire Ln."

A hash can be useful in situations where one object has many different properties with values or to store values for many different objects.
